Lógica de Simulación de Eventos

Diagramas de flujo detallados para los 5 problemas de la guía.

P1: Sistema M/M/1 Estándar

graph TD Start([Inicio]) --> Init[Reloj=0, Cola=0, PS=Libre] Init --> FEL[Programar Primera Llegada] FEL --> Loop{Extraer Próximo Evento} Loop -- LLEGADA --> NextArr[Programar Siguiente Llegada] NextArr --> CheckPS{¿PS Libre?} CheckPS -- Si --> Occupy[PS=1, Programar Fin Servicio] CheckPS -- No --> Enqueue[Cola++] Loop -- FIN_SERVICIO --> FreePS[PS=0] FreePS --> CheckQ{¿Cola > 0?} CheckQ -- Si --> Dequeue[Cola--, PS=1, Programar Fin Servicio] CheckQ -- No --> Loop Occupy --> Loop Enqueue --> Loop Dequeue --> Loop

P2: Servidor con Descansos y Pausa

graph TD Loop{Evento} -- SALIDA_SRV --> SrvOut[Servidor=Ausente] SrvOut --> CheckBusy{¿PS Ocupado?} CheckBusy -- Si --> Pause[Calcular Tiempo Restante
Eliminar FIN_SERVICIO de FEL] CheckBusy -- No --> Loop Loop -- LLEGADA_SRV --> SrvIn[Servidor=Presente] SrvIn --> CheckRem{¿Hay Remanente?} CheckRem -- Si --> Resume[Programar FIN_SERVICIO
con Tiempo Restante] CheckRem -- No --> CheckWait{¿Cola > 0?} CheckWait -- Si --> StartS[Atender Siguiente] Loop -- FIN_SERVICIO --> Done[PS=Libre, Remanente=0] Done --> CheckNext{¿Servidor Pres. Y Cola > 0?} CheckNext -- Si --> Next[Atender Siguiente]

P3: Abandono de Cola (Impaciencia)

graph TD Loop{Evento} -- LLEGADA --> Busy{¿PS Ocupado?} Busy -- Si --> Wait[Cola++, Guardar Hora Arribo
Programar ABANDONO en T+Limite] Busy -- No --> Serve[Atender] Loop -- FIN_SERVICIO --> NextInQ[Tomar de Cola] NextInQ --> CancelAb[Eliminar ABANDONO de FEL
para ese cliente] Loop -- ABANDONO --> CheckInQ{¿Cliente sigue en cola?} CheckInQ -- Si --> Leave[Cola-- / Sale del Sistema] CheckInQ -- No --> Ignore[Ignorar]

P4: Prioridades (A sobre B)

graph TD Loop{Evento} -- LLEGADA_A --> BusyA{¿PS Libre?} BusyA -- Si --> ServeA[PS=1, Programar FIN_SERVICIO] BusyA -- No --> EnqA[Cola A++] Loop -- LLEGADA_B --> BusyB{¿PS Libre?} BusyB -- Si --> ServeB[PS=1, Programar FIN_SERVICIO] BusyB -- No --> EnqB[Cola B++] Loop -- FIN_SERVICIO --> CheckA{¿Cola A > 0?} CheckA -- Si --> TakeA[Cola A--, PS=1, Programar FIN_SERVICIO] CheckA -- No --> CheckB{¿Cola B > 0?} CheckB -- Si --> TakeB[Cola B--, PS=1, Programar FIN_SERVICIO] CheckB -- No --> Free[PS=0]

P5: Zona de Seguridad (Traslado)

graph TD Loop{Evento} -- LLEGADA --> CheckSafe{¿Zona Libre y PS Libre?} CheckSafe -- Si --> Move[Zona=Ocupada, Programar LLEGADA_PS] CheckSafe -- No --> Enq[Cola++] Loop -- LLEGADA_PS --> AtPS[Zona=Libre, PS=Ocupada, Programar FIN_SERVICIO] Loop -- FIN_SERVICIO --> FreePS[PS=Libre] FreePS --> CheckQ{¿Cola > 0?} CheckQ -- Si --> NextMove[Cola--, Zona=Ocupada, Programar LLEGADA_PS] CheckQ -- No --> Loop
Simbología: Los elipses son inicio/fin, los rectángulos son procesos y los rombos son decisiones.
